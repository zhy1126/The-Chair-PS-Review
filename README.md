# The-Chair-PS-Review
面向 Codex 的留学 PS 定性诊断 Skill：判断文书是在匹配项目，还是重复 CV；聚焦主线逻辑、思考深度、经历取舍与项目契合，提供诊断和引导。（小红书：The椅子）

## 核心理念

- **契合优先于优秀展示**：PS 不是 GPA、奖项和名校经历的重复陈列。
- **深度优先于广度**：少量被充分展开的经历，通常比经历清单更有说服力。
- **主线优先于履历顺序**：时间顺序不等于逻辑，段落必须共同推进一个核心问题。
- **思考优先于语言包装**：流畅英语不能替代真实的判断、方法、证据与反思。
- **具体优先于项目宣传**：课程和资源必须回应申请者已经形成的训练需求。
- **真实优先于完美故事**：不制造转折，不夸大职责，不把回顾性理解写成虚假的因果。

## 它可以诊断什么

- 文书是在论证项目契合，还是主要证明申请者“很优秀”；
- 全文是否存在可以被一句话概括的核心主线；
- 每段经历是否承担独特的论证功能；
- 是否出现 GPA、奖项、机构和任务的 CV 式罗列；
- 经历中是否真正呈现了问题、推理、证据、局限和认识变化；
- 项目资源是否形成“过去经历—训练需求—项目资源—未来用途”的完整链条；
- 因果关系、时间顺序和申请者职责是否可信；
- 哪些内容应该保留、深化、压缩、合并、移动、删除或重构。

## 输出内容

一次完整诊断通常包括：

1. 测评依据与信息边界；
2. 一句话定性结论；
3. 当前可识别的契合主张；
4. 结构性、主要和局部风险；
5. 五个维度的定性诊断；
6. 逐段主线图；
7. CV 与 PS 审查；
8. 项目匹配审查；
9. 三项最高优先级修改任务；
10. 需要申请者本人回答的问题；
11. 自我修改清单。

本 Skill **默认不提供数字总分**。不同项目、题目和字数限制对优秀文书的要求并不相同，数字不能替代基于文本证据的判断。

## 不会做什么

- 不输出可直接提交的完整 PS；
- 不替申请者重写整段或整篇文书；
- 不编造经历、反思、职业目标或项目资源；
- 不为了让故事流畅而制造虚假因果；
- 不把录取结果直接等同于文书质量；
- 不在缺少项目和题目背景时假装完成了项目匹配判断。

## 适用范围

当前版本主要用于研究生留学申请中的：

- Personal Statement；
- Statement of Purpose；
- Statement of Academic Purpose；
- Programme-specific motivation essay；
- 短篇 academic/professional interest statement。

现有校准案例最适合英国、中国香港和新加坡的授课型硕士申请，覆盖法律、社会科学、公共政策、人力资源管理和商业分析等方向。

它不应被直接套用于本科主文书、博士研究计划、奖学金文书、推荐信。这些材料需要不同的评价任务和结构标准。

## 使用前需要准备

为避免脱离题目评价文书，请尽量提供完整申请语境：

```text
请使用 $review-study-abroad-ps 对我的文书进行定性诊断。
请只诊断和引导，不代写成稿，也不要默认给数字评分。

【申请类别】授课型硕士 / 研究型硕士 / 本科 / 博士 / 其他：
【文书类型】PS / SOP / Statement of Academic Purpose / Why Programme / Short Answer / 其他：
【申请地区】
【目标学校】
【目标项目（完整名称）】
【项目官网】请提供官方链接；如无法访问，请粘贴官方项目与课程信息：
【官方文书题目或要求】请完整粘贴：
【字数或字符限制】
【当前字数】
【当前版本】初稿 / 修改稿 / 定稿前：

【申请者背景】上传 CV，或简要列出教育、相关经历和必要背景：
【希望建立的方向】尚未确定也可以如实说明：
【希望重点检查】定位 / 主线 / 经历取舍 / 项目匹配 / 真实性 / 压缩 / 其他：

【文书正文】上传文件或粘贴全文：
```

如果缺少官方题目、项目或字数限制，Skill 仍可进行正文预检，但会将无法判断的部分明确标为“信息不足”，而不是把缺失信息当作文书问题。

## 安装

下载或克隆本仓库，将 Skill 目录复制到个人 Codex skills 目录：

```text
$CODEX_HOME/skills/review-study-abroad-ps/
```

当 `CODEX_HOME` 未单独设置时，个人目录通常为：

```text
~/.codex/skills/review-study-abroad-ps/
```

确保该目录直接包含：

```text
SKILL.md
agents/
references/
scripts/
```

安装后，在 Codex 中使用当前 Skill ID 调用：

```text
$review-study-abroad-ps
```

## 推荐仓库结构

仓库级 README 与运行时 Skill 分开存放：

```text
the-chair-ps-review/
├── README.md
├── LICENSE
└── skill/
    └── review-study-abroad-ps/
        ├── SKILL.md
        ├── agents/
        ├── references/
        └── scripts/
```

这样 README 用于 GitHub 展示，不会占用 Skill 运行时上下文。

## 隐私与案例

公开仓库不得包含客户原始文书、姓名、申请编号、联系方式或其他可以识别申请者的信息。

本项目只应发布：

- 经匿名化和合成处理的案例模式；
- 不包含原文复现的判断规则；
- 可公开使用的评价框架和工具脚本。

如果提交 Issue 或参与校准，请不要直接上传真实申请材料。可以提供脱敏后的片段、问题类型和你对诊断结果的反馈。

## 项目定位

The Chair 不是招生委员会，也不代表任何大学或项目。本项目提供的是一种以项目契合、逻辑结构和真实思考为中心的文书诊断方法，不保证录取结果。
