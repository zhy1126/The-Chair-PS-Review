# Application Contexts and Intake

## Primary scope

Use this skill primarily for postgraduate taught-programme applications, including:

- Personal Statement (PS)
- Statement of Purpose (SOP)
- Statement of Academic Purpose
- Programme-specific motivation essays
- Short academic or professional interest statements

The calibration set is strongest for UK, Hong Kong, and Singapore master's applications in law, social science, policy, HRM, and business analytics.

Do not silently apply this rubric to undergraduate Common App personal essays, Personal History Statements, doctoral research proposals, recommendation letters, or scholarship essays. Identify the different task first and explain which parts of the rubric remain applicable.

## Required intake

Use `intake-template.md`. Treat these as required for a complete programme-specific diagnosis:

- Draft text or document.
- Application category and essay type.
- Exact university and programme.
- Official programme page or pasted official programme information.
- Exact essay prompt or application instruction.
- Word limit.

Treat the CV or background inventory, intended direction, and draft stage as recommended context. They improve experience-selection and credibility checks but are not always required.

## Missing-information behaviour

When required context is missing, state:

1. What was provided.
2. What is missing.
3. Which conclusions are provisional.
4. What the missing material would allow the reviewer to test.

Then either ask the user to complete the fixed intake template or, if the user wants immediate feedback, perform a provisional text-only precheck. Do not issue a final programme-match verdict or numerical score from incomplete context. Ask no more than three high-value follow-up questions at once.

## Word-limit behaviour

Treat 750-1000 words as a constrained argument. Two or three developed experiences are a useful default. Adjust for the prompt rather than enforcing a fixed paragraph count.

For short answers, evaluate whether each sentence directly serves the prompt. For multi-essay applications, prevent duplicated stories unless each use reveals a genuinely different dimension.

## DOCX behaviour

When a DOCX contains comments or revisions, inspect them because they may reveal unresolved factual, logical, or authorship questions. Use `scripts/extract_docx.py` if native document reading does not expose them.
